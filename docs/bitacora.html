Configuración del entorno de desarrollo

Setup

Se instaló MySQL 8.0 en el entorno WSL2 (Ubuntu 24.04) del equipo de desarrollo. Se configuró el usuario root con contraseña para acceso local y se creó la base de datos juego_piurano_db con la tabla tabla_records según el esquema proporcionado por el manual del Dr. Paiva Ayala.

  • Instalación: sudo apt-get install mysql-server
  • Creación de base de datos y tabla mediante sentencias SQL del manual
  • Instalación del MCP server @benborla29/mcp-server-mysql para integración con el entorno de desarrollo
  • Configuración de variables de entorno: host 127.0.0.1, puerto 3306, base juego_piurano_db

Diseño de arquitectura y planificación SDD

Arquitectura

Se definió la arquitectura cliente-servidor del juego siguiendo el modelo desacoplado requerido por el manual. Se eligió Node.js con Express para el backend por su compatibilidad con el entorno de desarrollo y por simplificar la gestión asíncrona de la conexión MySQL.

  • Frontend: HTML5 Canvas (800×600), CSS3, JavaScript vanilla
  • Backend: Node.js + Express + mysql2 con prepared statements
  • Patrón de estado central con máquina de estados: inicio → nivel1 → nivel2 → gameover
  • Convención de código: funciones y variables en español, sin comentarios
  • Se definieron los módulos: juego.js, nivel1.js, nivel2.js, api.js

Implementación del juego completo

Implementación

Se desarrollaron todos los archivos del proyecto: 9 archivos, 2075 líneas de código. Se implementaron ambos niveles con mecánicas diferenciadas, el backend con persistencia MySQL y el formulario de ingreso de nombre.

  • Nivel 1: 4 panales con barra de agresividad, 20 ráfagas de humo, temporizador 60s, efecto visual de humo, abejas animadas orbitando
  • Nivel 2: Velocímetro de RPM con arco Canvas, zona óptima 600–1000 RPM, barra de extracción de miel, rotación animada de centrífuga
  • Backend: endpoint POST /api/guardar-record con validación y prepared statements
  • Sistema de puntaje: inicio 1000 pts, +10 por panal pacificado, +200 al completar nivel 2, -1 pt/segundo
  • Commit inicial: da12fbb
SESIÓN 4 — 10 de junio de 2026

Correcciones y mejoras de jugabilidad

Bug Fix Mejora

Se identificaron y corrigieron varios problemas de jugabilidad reportados durante las pruebas.

  • Bug: extracción de miel demasiado lenta. Tasa original: 0.3%/s (requería 333s para 100%). Corregida a 1.5%/s (completable en ~65s con el timer de 90s).
  • Mejora: controles de centrífuga. Fricción reducida de 5 RPM/frame a 1 RPM/frame. El jugador ya no necesita mantener la tecla presionada; alcanza la zona óptima y el RPM cae lentamente por inercia.
  • Bug: click en cualquier lugar eliminaba abejas. Se agregó hit-testing por rectángulo con margen de 25px. Ahora el click debe apuntar al panal para tener efecto.
  • Bug: árbol superpuesto al formulario de inicio. El árbol decorativo central fue movido a los costados (x=110 y x=690). Se extrajo función dibujarArbolAlgarrobo() reutilizable.
  • Bug: label "Ingresá tu nombre" aparecía detrás del formulario HTML. Se migró el label del canvas al div HTML superpuesto como elemento <p>.
  • Bug: superposición del formulario de inicio sobre la pantalla de gameover. Corregido con removerDivSuperpuesto() al inicio de finalizarJuego() y dibujar() en reiniciarJuego().
