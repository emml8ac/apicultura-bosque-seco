1. Descripción del Juego

Apicultura en el Bosque Seco es un videojuego educativo de 2 niveles ambientado en la región Piura, Perú. El jugador asume el rol de un apicultor del bosque seco que debe calmar los panales de abejas y operar una centrífuga para extraer la miel.

El juego pone a prueba la precisión y el control del jugador mientras presenta elementos de la apicultura regional piurana.

2. Requisitos del Sistema

Para jugar (modo en línea)

  • Navegador web moderno: Chrome 90+, Firefox 88+, Edge 90+ o Safari 14+
  • Ratón o trackpad (Nivel 1 requiere clicks)
  • Teclado (Nivel 2 requiere teclas A / D)
  • Conexión a internet (para cargar la versión de GitHub Pages)

Para jugar con persistencia de récords (modo local)

  • Node.js 18 o superior
  • MySQL 8.0 o superior
  • npm 9 o superior
  • Sistema operativo: Windows 10/11, macOS 12+, o Ubuntu 22+

3. Instalación y Ejecución

3.1 Clonar el repositorio

git clone https://github.com/emml8ac/apicultura-bosque-seco.git
cd apicultura-bosque-seco

3.2 Configurar la base de datos MySQL

Crear la base de datos y la tabla ejecutando las siguientes sentencias SQL en el cliente MySQL:

CREATE DATABASE IF NOT EXISTS juego_piurano_db;
USE juego_piurano_db;

CREATE TABLE IF NOT EXISTS tabla_records (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nombre_jugador VARCHAR(100) NOT NULL,
  puntaje_total INT NOT NULL,
  tiempo_segundos INT NOT NULL,
  nivel_alcanzado TINYINT NOT NULL,
  fecha_registro TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

3.3 Instalar dependencias del backend

cd backend
npm install

3.4 Iniciar el servidor

node server.js
# El servidor inicia en http://localhost:3000

3.5 Abrir el juego

Abrí el archivo index.html con el servidor Live Server de VS Code (puerto 5500) o cualquier servidor HTTP local. No abrir directamente como archivo (file://) ya que las peticiones fetch quedarán bloqueadas por CORS.

Tip: Con VS Code, instalar la extensión "Live Server" y hacer click derecho en index.html → "Open with Live Server". El juego abre en http://127.0.0.1:5500.

4. Cómo Jugar

4.1 Pantalla de inicio

Al abrir el juego se muestra la pantalla de inicio con el título y un formulario de ingreso de nombre. Escribí tu nombre y hacé click en Iniciar (o presioná Enter) para comenzar.

Nota: El nombre puede tener hasta 50 caracteres. No puede quedar vacío.

4.2 Nivel 1 — Pacificación de Panales

Objetivo: Reducir la agresividad de los 4 panales a 0 antes de que se acabe el tiempo (60 segundos) o las ráfagas de humo (20 ráfagas).

Controles

Click
→ sobre un panal para aplicar una ráfaga de humo

Cada click sobre un panal reduce su agresividad en 25 puntos. Se necesitan 4 ráfagas por panal para pacificarlo completamente.

Mecánicas clave

  • Los 4 panales tienen una barra de agresividad (naranja → verde).
  • El click debe apuntar directamente al panal con un margen de ±25 px. Hacer click en el aire no tiene efecto.
  • Puntaje: +10 puntos por cada panal pacificado.
  • Al pacificar los 4 panales, el juego avanza automáticamente al Nivel 2.
  • Si se acaban las ráfagas o el tiempo, el juego termina.
Atención: El humo NO actúa si hacés click fuera del área del panal. Apuntá bien.

4.3 Nivel 2 — Operación de la Centrífuga

Objetivo: Mantener la centrífuga en la zona óptima de RPM para extraer el 100% de la miel antes de que se acabe el tiempo (90 segundos).

Controles

D
→ Acelerar la centrífuga (+10 RPM por frame)
A
→ Desacelerar la centrífuga (-20 RPM por frame)
← →
→ Flechas del teclado (mismo efecto que A/D)

Zonas del velocímetro

ZonaRPMColorEfecto
Sin extracción0–599GrisNo extrae miel
Óptima600–1000VerdeExtrae miel activamente
Peligrosa+1001RojoPierde puntos continuamente

Mecánicas clave

  • Al soltar la tecla, la centrífuga frena lentamente (-1 RPM/frame) por inercia.
  • La extracción es proporcional al RPM dentro de la zona óptima.
  • Al llegar al 100% de extracción, el juego finaliza con victoria y se guarda el récord.
  • Si el tiempo llega a 0 antes del 100%, el juego termina sin guardar récord.
Estrategia: Acelerá con D hasta unos 750–800 RPM, soltá la tecla y dejá que la inercia mantenga los RPM en la zona verde. Así no gastás energía manteniendo presionada la tecla.

El récord solo se guarda en la base de datos si el jugador completa el Nivel 2 (extracción al 100%).

6. Panel de Administración

Para ver todos los récords almacenados en la base de datos, ingresá el siguiente nombre en el formulario de inicio:

AdministradorUNP435*

Se mostrará una tabla con todos los récords guardados: nombre del jugador, puntaje, tiempo en segundos, nivel alcanzado y fecha.

Requisito: El panel de administración solo funciona si el backend Node.js está corriendo en localhost:3000.

7. Solución de Problemas

El juego no guarda récords

Verificá que el servidor backend esté activo (node backend/server.js) y que MySQL esté corriendo. La consola del navegador (F12) mostrará el error exacto de conexión.

La centrífuga no responde a las teclas

Hacé click una vez sobre el lienzo del juego para asegurarte de que tiene el foco del teclado. Si el cursor está sobre otro elemento de la página las teclas no llegarán al canvas.

El juego carga en blanco

Asegurate de estar accediendo desde un servidor HTTP (Live Server, localhost) y no desde file://. El protocolo file bloquea fetch() por seguridad en todos los navegadores modernos.
